## 🔥 Firebase vs Supabase — що краще для Knitly?

### TL;DR: **Залишайся на Firebase.** Міграція зараз — це місяць роботи без нових фіч.

### Порівняння Free тієрів:

| Параметр | Firebase (Spark) | Supabase (Free) |
|---|---|---|
| **БД Storage** | 1 GB Firestore | 500 MB PostgreSQL |
| **Reads/день** | 50,000 | Необмежено (але є ліміти CPU) |
| **Writes/день** | 20,000 | Необмежено |
| **File Storage** | **5 GB** | 1 GB |
| **Bandwidth** | 10 GB/місяць | **Необмежено** |
| **Auth** | Необмежено | Необмежено |
| **Real-time** | Вбудований (Firestore) | 200 конект. (Realtime) |
| **Functions** | 125k викликів/міс | 500k викликів/міс |
| **Пауза проєкту** | Ніколи | **Після 1 тижня бездіяльності!** |

### Критичний момент для музичної платформи:
- **Аудіо-файли** — це головна стаття витрат
- Firebase Storage: 5 GB безкоштовно, потім $0.026/GB зберігання + **$0.12/GB download** (дорого!)
- Supabase Storage: 1 GB безкоштовно, потім $0.021/GB зберігання + **необмежений bandwidth**

### Моя рекомендація:
1. **Зараз**: Лишайся на Firebase — реальний час (чат, сповіщення) там значно краще
2. **Для файлів**: Розглянь **Cloudflare R2** для аудіо ($0/GB bandwidth — безкоштовно!)
3. **Коли буде >100 активних юзерів**: Firebase Blaze (pay-as-you-go) — просто увімкни картку, перших місяців вистачить безкоштовного ліміту
4. **Коли буде >1000 юзерів**: Тоді вже думати про архітектуру і Supabase для аналітики

## 🐛 Знайдені баги та проблеми

### КРИТИЧНІ (треба виправити перед запуском)

#### 1. Заморожування навігації (Navigation Freeze)
**Де:** `Home.js` → `Feed.jsx`
**Причина:** Feed рендерить пости без debounce при навігації. `useWhyDidYouUpdate` hook — налагоджувальний хук — запущений в продакшні, він додає логи і навантаження.
**Рішення:** Видалити `useWhyDidYouUpdate` з production builds + додати `Suspense` + `React.lazy` для важких компонентів.

#### 2. Feed показує лише перших 10 підписок
**Де:** `Feed.jsx:23` — `followingList.slice(0, 10)`
**Причина:** Firebase обмежує `array-contains-any` до 10 елементів — коментар в коді є, але проблема не вирішена.
**Рішення:** Розбити запит батчами по 10, мержити результати (код вже є в `recommendationService.js`!).

#### 3. `recommendationService.js` — написаний але **ніде не використовується**
**Де:** `src/services/recommendationService.js`
**Статус:** Файл є, логіка правильна, але ніде не імпортується.
**Рішення:** Підключити до `Feed.jsx` замість поточного простого запиту.

#### 4. ShareModal стан — дублювання
**Де:** `App.js` і `Home.js` — обидва мають власний стан `isShareModalOpen`
**Причина:** Локальний стан у Home.js не пов'язаний з глобальним в App.js
**Рішення:** Видалити локальний стан в Home.js, використовувати тільки пропс `openShareModal` з App.js.

#### 5. Відсутня адаптивність на деяких сторінках
**Де:** `LeftSidebar.css` — sidebar просто зникає на мобільному, без альтернативи
**Рішення:** Переконатись що Bottom NavBar завжди видимий і покриває всі пункти навігації сайдбару.

### СЕРЕДНІ (треба виправити)

#### 6. Debug hook у продакшні
**Де:** `Home.js:2`, `Feed.jsx:9` — `import useWhyDidYouUpdate`
**Рішення:** Видалити ці рядки або обгорнути в `if (process.env.NODE_ENV === 'development')`.

#### 7. Firebase credentials у клієнтському коді
**Де:** `src/services/firebase.js`
**Ризик:** Технічно для Firebase це нормально (ключі захищені Rules), але краща практика — Environment Variables.
**Рішення:** Перенести в `.env.local`, додати обмеження домену в Firebase Console.

#### 8. Стрічка не оновлюється в реальному часі
**Де:** `Feed.jsx` — використовує `getDocs` (одноразовий запит) замість `onSnapshot`
**Рішення:** Для головної стрічки — додати `onSnapshot` або кнопку "Нові дописи" з re-fetch.

#### 9. Відсутня пагінація в Feed
**Де:** `Feed.jsx` — `limit(10)` без можливості завантажити більше
**Рішення:** Додати кнопку "Завантажити ще" або infinite scroll з `startAfter()`.

#### 10. `formatRelativeTime` — неправильні українські відмінювання
**Де:** `Home.js:30` — `"${diffInMinutes} хвилин${...}"` — логіка неповна
**Приклад:** "2 хвилини" — правильно, але "5 хвилин" теж правильно (без 'и')
**Рішення:** Використати повну логіку відмінювання (1 → 'хвилина', 2-4 → 'хвилини', 5+ → 'хвилин').

### КОСМЕТИЧНІ

#### 11. "Repost скоро™" — placeholder в продакшні
**Де:** `PostCard.jsx` — кнопка репосту показує toast "незабаром"
**Рішення:** Або реалізувати репост, або приховати кнопку до реалізації.

#### 12. Magic numbers скрізь
- `limit(50)` для тегів в Home.js
- `limit(10)` для постів
- `100ms` затримка для Feed
Рішення: Винести в `src/config/constants.js`.

## 💡 Мої ідеї та поради

### 1. Стратегія ребрендингу дизайну

**Поточний дизайн:** Темна тема, glassmorphism, фіолетово-рожеві градієнти.
**Проблема:** Схожий на 100 інших darkmode SaaS — не запам'ятовується.

**Пропозиція для Knitly:**
- **Головний колір:** Не фіолетовий (занадто "tech"), а **теплий amber/gold** (#F59E0B) + темний фон (#0D0D0D) — асоціюється з "живою музикою", вогнем, емоціями
- **Акцент:** Лишити один фіолетовий акцент для интерактивних елементів
- **Типографіка:** Додати більший font-size для назв треків — вони мають "дихати"
- **Картки треків:** Збільшити cover art — на SoundCloud воно маленьке, тут зроби більшим
- **Анімації:** Менше glassmorphism, більше "живих" ефектів (наприклад, звукова хвиля як фон плеєра)

### 2. Критична фіча для утримання — "Зараз слухаю"
Ідея вже є в ideas.md (#22). Реалізуй **першою**:
- Кнопка "Поділитись" прямо в плеєрі
- Автоматично створює пост "Зараз слухаю: [трек]" з прогрес-баром
- Це вірусна механіка — інші бачать і хочуть теж послухати

### 3. Онбординг — першочергово
**Проблема:** Новий юзер заходить і бачить порожній feed (бо нема підписок).
**Рішення — 3 кроки:**
1. При реєстрації → mini-опитування (5 питань про жанри)
2. Одразу показати 10 артистів у відповідних жанрах → "підпишись на цих"
3. Перший feed — суміш популярного + за жанрами

### 4. PWA замість нативних аппок (на початку)
**Замість React Native** (дорого, довго) — зроби спершу **PWA**:
- `manifest.json` + Service Worker
- Push-нотифікації через Firebase Cloud Messaging
- Офлайн-кеш для вже прослуханих треків
- "Встанови додаток" банер
- Це займе 2-3 дні, нативні аппки — 3-6 місяців

### 5. Пошук — підключи Algolia або Typesense
**Поточний стан:** Пошук через Firestore `searchKeywords` — дуже обмежений.
**Algolia Free:** 10k операцій/місяць — достатньо для старту.
**Typesense (self-hosted):** Безкоштовно, але потребує сервер.
**Що дає:** Повнотекстовий пошук, typo-tolerance ("ройял" → "Royal Blood"), faceted search (по жанрах).

### 6. Аудіо-стрімінг — використай Firebase Storage правильно
**Поточна проблема:** Кожне прослуховування — це окремий download з Firebase Storage (дорого).
**Рішення:**
- Додати HTTP range requests (вже підтримується в Firebase Storage)
- Кешувати в браузері через Service Worker
- Для популярних треків — CDN перед Firebase (Cloudflare free plan)

### 7. Пріоритет фіч (на мій погляд)

```
ЗАРАЗ (до запуску):
├── Виправити navigation freeze
├── Виправити Feed pagination
├── Підключити recommendationService
├── Онбординг для нових юзерів
└── Кнопка "Нові дописи" в стрічці

ФАЗА 1 (перші 2 місяці після запуску):

├── "Зараз слухаю" пост
├── Реальний пошук (Algolia)
├── Покращений мобільний дизайн

ФАЗА 2 (після першої аудиторії):
├── Artist Dashboard повноцінний
├── Система подарунків/Ноти
├── Колаборативні плейлисти

ПІЗНІШЕ:
├── Кімнати для спільного прослуховування

```

### 8. Монетизація — моє бачення

**Не починай з "Нот" одразу** — складна у пояснені нова концепція.
**Починай з простого:**
1. **Артист Pro** ($5/місяць) — розширена аналітика, верифікована позначка, пріоритет в рекомендаціях
2. **Пряма підтримка** — кнопка "Підтримати артиста" → пряме посилання на Monobank/PayPal/Buy Me a Coffee
3. **Промо** — артист платить за просування треку в стрічці інших

"Ноти" — хороша ідея, але це цілий продукт всередині продукту. Спочатку валідуй модель.

### 9. Вирішення "проблеми курки та яйця"

З Globaly-ideas-project.md це правильно зазначено. Моя стратегія:
1. **Instagram/TikTok DM кампанія** — знайти 20 укр. інді-артистів, написати особисто
2. **Перша хвиля** — запросити їх підписників (кожен отримує 10 інвайтів)
3. **Контент від команди** — перші тижні команда сама постить огляди треків
4. **Партнерство** — написати в СЛУХ, Neformat, LiRoom

### 10. Технічні покращення

**Додати Error Boundaries:**
```jsx
// Загорнути Feed та інші важкі компоненти
<ErrorBoundary fallback={<div>Щось пішло не так. Оновіть сторінку.</div>}>
  <Feed ... />
</ErrorBoundary>
```

**React.lazy для важких сторінок:**
```jsx
const MessagesPage = React.lazy(() => import('./pages/MessagesPage'));
const Dashboard = React.lazy(() => import('./pages/Dashboard'));
// Це зменшить початковий bundle і прискорить завантаження
```

**Видалити з продакшну:**
- `useWhyDidYouUpdate` hook (2 місця)
- `console.log` та `console.error` (замінити на production logger)

1. **Спершу зупинити баги** (navigation freeze, feed pagination)
2. **Потім дизайн** — починай з кольорової палітри та Typography scale
3. **Мобільний адаптив** — перевірити кожну сторінку на 375px (iPhone SE)
4. **Онбординг** — без нього новий юзер іде через 30 секунд
